# 内购商品展示平台 — 同事操作手册

> **阅读对象**：完全不懂编程的同事
> **你要做的事**：注册两个免费网站的账号，复制粘贴一些内容，最后得到一个属于你自己的商品展示网站
> **预计耗时**：40-60 分钟（第一次会慢一些，别着急）
> **需要准备**：一个邮箱、一台电脑、Chrome 浏览器

---

## 整体流程一览

整个过程分 **5 大步**，按顺序做就行：

```
第一步：注册 GitHub 账号 → 把代码传上去
第二步：注册 Supabase 账号 → 创建你的数据库项目
第三步：在 Supabase 里建数据库表（复制粘贴 SQL）
第四步：在 Supabase 里建图片仓库
第五步：把你的 Supabase 信息填到代码里 → 开启 GitHub Pages 发布网站
```

做完这 5 步，你就有了自己的商品展示网站。下面开始！

---

# 第一步：注册 GitHub 并上传代码

> GitHub 是一个存放代码的网站，你可以理解为"代码网盘"。

## 1.1 注册 GitHub 账号

1. 打开浏览器，进入 **https://github.com**
2. 点击右上角 **Sign up**（注册）
3. 依次填写：
   - **邮箱**：填你的邮箱
   - **密码**：设一个密码
   - **用户名**：起一个英文名（比如 zhangsan-work）
4. 按页面提示完成人机验证
5. 去你的邮箱，找到 GitHub 发来的验证邮件，点击里面的链接完成验证
6. 验证完成后，回到 GitHub 页面，你就注册好了

## 1.2 创建一个代码仓库

1. 登录 GitHub 后，点击页面右上角的 **+** 号 → 选择 **New repository**
2. 在页面中填写：
   - **Repository name** 那一栏填：`product-showcase`
   - 下面的 **Public** 保持选中（默认就是）
   - ⚠️ **不要勾选** 下面的 "Add a README file"
3. 点击绿色按钮 **Create repository**

## 1.3 上传代码文件

> 我给你的文件夹叫 `product-showcase`，里面有这些东西：
> - `index.html`、`admin.html`
> - `css` 文件夹（里面有 `style.css`）
> - `js` 文件夹（里面有 4 个文件）
> - `sql` 文件夹（里面有 1 个文件）

1. 在刚才创建的仓库页面，你会看到一段提示文字，找到 **uploading an existing file** 这个蓝色链接，点击它
2. 打开你电脑上的 `product-showcase` 文件夹
3. **进入文件夹内部**，全选里面的所有文件和文件夹（Ctrl+A）
4. 把它们**拖拽**到浏览器页面的上传区域

> ⚠️ **特别注意**：是把文件夹**里面的东西**拖进去，不是把文件夹本身拖进去！
>
> ✅ 正确：拖进去后看到 `index.html`、`admin.html`、`css/`、`js/` 等
>
> ❌ 错误：拖进去后只看到一个 `product-showcase/` 文件夹

5. 等文件全部上传完成后（进度条走完）
6. 页面下方有个 **Commit changes** 按钮，直接点击它
7. 等待几秒，上传完成！

## 1.4 确认上传成功

上传完成后，你的仓库页面应该能看到这些文件：

```
index.html
admin.html
css/
js/
sql/
```

如果看到的是这样，说明上传正确 ✅

# 第二步：注册 Supabase 并创建项目

> Supabase 是一个免费的在线数据库服务。你可以理解为"云端的 Excel 表格 + 图片仓库"。

## 2.1 注册 Supabase 账号

1. 打开浏览器，进入 **https://supabase.com**
2. 点击页面上的 **Start your project** 或右上角的 **Sign Up**
3. 选择 **Continue with GitHub**（用你刚注册的 GitHub 账号登录，最方便）
4. 在弹出的页面中点击 **Authorize supabase**（授权）
5. 登录成功！

## 2.2 创建新项目

1. 登录后，点击 **New Project**（新建项目）
2. 填写以下内容：
   - **Organization**：默认的就行，不用改
   - **Project name**：填 `product-showcase`
   - **Database Password**：设一个密码（比如 `MyDb2024!`）
     - ⚠️ **这个密码一定要记下来**，写在记事本里
   - **Region**：选择 **Northeast Asia (Tokyo)**（东京，离我们最近）
3. 点击 **Create new project**
4. 等待 1-2 分钟，页面会显示项目正在创建中
5. 创建完成后会自动跳转到项目主页

## 2.3 找到并记录两个重要信息

> 接下来你需要找到两个东西，后面要用到。请复制后粘贴到记事本里保存。

1. 在页面左侧菜单栏，找到最下面的 **⚙️ Settings**（齿轮图标），点击它
2. 在展开的子菜单中，点击 **API**
3. 现在你能看到两个重要信息：

**第一个：Project URL**
- 在页面上方，标题写着 "Project URL"
- 下面有一个类似 `https://xxxxxxxx.supabase.co` 的网址
- 点击右边的复制按钮，粘贴到记事本里
- 在前面标注：**项目地址**

**第二个：anon public key**
- 往下看，标题写着 "Project API keys"
- 找到 **anon** 那一行（旁边写着 public）
- 点击右边的复制按钮，粘贴到记事本里
- 在前面标注：**公钥**

> ⚠️ **注意**：下面还有一个 **service_role** 的 key，**不要复制那个**！只复制 anon 那个。

现在你的记事本里应该有这样的内容：

```
项目地址：https://xxxxxxxx.supabase.co
公钥：eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxxxx很长一串xxxxx
```

# 第三步：创建数据库表

> 这一步你需要在 Supabase 里执行三段"命令"，就是复制粘贴然后点运行，不需要理解内容。

## 3.1 打开 SQL 编辑器

1. 在 Supabase 左侧菜单栏，找到 **SQL Editor**（图标像一个终端窗口），点击它
2. 点击 **New query**（新建查询）

## 3.2 创建第一个表：商品表

1. 把下面这段内容**完整复制**（从第一行到最后一行）：

```sql
CREATE TABLE products (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT NOT NULL,
  price NUMERIC(10, 2) NOT NULL,
  sku TEXT DEFAULT '',
  quantity INTEGER DEFAULT 0,
  product_sku TEXT DEFAULT '',
  remark TEXT DEFAULT '',
  image_url TEXT NOT NULL,
  status TEXT DEFAULT 'active' CHECK (status IN ('active', 'inactive')),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE products ENABLE ROW LEVEL SECURITY;

CREATE POLICY "allow_anon_select_products"
  ON products FOR SELECT TO anon USING (true);

CREATE POLICY "allow_anon_insert_products"
  ON products FOR INSERT TO anon WITH CHECK (true);

CREATE POLICY "allow_anon_update_products"
  ON products FOR UPDATE TO anon USING (true) WITH CHECK (true);

CREATE POLICY "allow_anon_delete_products"
  ON products FOR DELETE TO anon USING (true);
```

2. 粘贴到 SQL Editor 的编辑区域（就是页面中间那个大的白色输入框）
3. 点击右下角的绿色 **Run** 按钮
4. 下方出现 **Success. No rows returned** 就表示成功了 ✅

> 如果出现红色错误提示 "relation products already exists"，说明这个表已经存在了。
> 这时候先在编辑器里输入 `DROP TABLE IF EXISTS products CASCADE;` 点 Run 删掉旧表，
> 然后再重新粘贴上面的内容执行一次。

## 3.3 创建第二个表：购买记录表

1. 点击 **New query**（再新建一个查询，不要在刚才那个里面继续写）
2. 把下面这段内容**完整复制**：

```sql
CREATE TABLE orders (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  product_id UUID NOT NULL REFERENCES products(id),
  product_name TEXT NOT NULL,
  buyer_name TEXT NOT NULL,
  buyer_ip TEXT DEFAULT '',
  quantity INTEGER NOT NULL CHECK (quantity > 0),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE orders ENABLE ROW LEVEL SECURITY;

CREATE POLICY "allow_anon_select_orders"
  ON orders FOR SELECT TO anon USING (true);

CREATE POLICY "allow_anon_insert_orders"
  ON orders FOR INSERT TO anon WITH CHECK (true);

CREATE POLICY "allow_anon_delete_orders"
  ON orders FOR DELETE TO anon USING (true);
```

3. 粘贴到编辑区域，点击 **Run**
4. 看到 **Success** 就成功了 ✅

## 3.4 创建购买功能

1. 再点击 **New query**
2. 把下面这段内容**完整复制**：

```sql
CREATE OR REPLACE FUNCTION purchase_product(
  p_product_id UUID,
  p_quantity INTEGER,
  p_buyer_name TEXT,
  p_buyer_ip TEXT DEFAULT ''
) RETURNS JSON AS $$
DECLARE
  v_product RECORD;
  v_order RECORD;
BEGIN
  SELECT id, name, quantity INTO v_product
  FROM products
  WHERE id = p_product_id
  FOR UPDATE;

  IF v_product.quantity < p_quantity THEN
    RAISE EXCEPTION 'insufficient_stock';
  END IF;

  UPDATE products
  SET quantity = quantity - p_quantity,
      updated_at = NOW()
  WHERE id = p_product_id;

  INSERT INTO orders (product_id, product_name, buyer_name, buyer_ip, quantity)
  VALUES (p_product_id, v_product.name, p_buyer_name, p_buyer_ip, p_quantity)
  RETURNING * INTO v_order;

  RETURN row_to_json(v_order);
END;
$$ LANGUAGE plpgsql;
```

3. 粘贴到编辑区域，点击 **Run**
4. 看到 **Success** 就成功了 ✅

## 3.5 确认表创建成功

1. 在左侧菜单栏，点击 **Table Editor**（表格图标）
2. 你应该能看到两个表：**products** 和 **orders**
3. 如果都能看到，说明数据库配置完成 ✅

> ⚠️ **这三段必须按顺序执行**！第二段依赖第一段，如果顺序搞反了会报错。

# 第四步：创建图片仓库

> 商品图片需要一个地方存放，这一步就是创建那个"图片仓库"。

## 4.1 创建存储桶

1. 在左侧菜单栏，点击 **Storage**（存储图标）
2. 点击 **New bucket**（新建存储桶）
3. 在弹出的窗口中：
   - **Name** 填写：`product-images`
     - ⚠️ 必须一字不差地填这个名字，包括中间的横杠
   - **Public bucket** 右边的开关：**打开它**（拨到右边，变成绿色/蓝色）
4. 点击 **Create bucket**

> ⚠️ **最容易出错的地方**：如果忘记打开 Public 开关，后面商品图片会显示不出来。
> 补救方法：点击 `product-images` 右边的三个点 `⋯` → **Edit bucket** → 打开 Public 开关 → 保存。

## 4.2 设置图片仓库的权限

> 创建完存储桶后，还需要告诉它"允许上传和查看图片"，否则网站无法正常使用。

1. 点击刚创建的 **product-images** 进入
2. 点击页面上方的 **Policies**（策略）标签
3. 你需要创建 **3 条策略**，一条一条来：

### 策略一：允许查看图片

1. 点击 **New policy**
2. 选择 **For full customization**
3. 填写：
   - **Policy name**：`allow_public_read`
   - **Allowed operation**：选择 **SELECT**
   - **Target roles**：选择 **anon**
   - **USING expression** 那个输入框里填：`true`
4. 点击 **Review** → 点击 **Save policy**

### 策略二：允许上传图片

1. 点击 **New policy**
2. 选择 **For full customization**
3. 填写：
   - **Policy name**：`allow_public_upload`
   - **Allowed operation**：选择 **INSERT**
   - **Target roles**：选择 **anon**
   - **WITH CHECK expression** 那个输入框里填：`true`
4. 点击 **Review** → 点击 **Save policy**

### 策略三：允许删除图片

1. 点击 **New policy**
2. 选择 **For full customization**
3. 填写：
   - **Policy name**：`allow_public_delete`
   - **Allowed operation**：选择 **DELETE**
   - **Target roles**：选择 **anon**
   - **USING expression** 那个输入框里填：`true`
4. 点击 **Review** → 点击 **Save policy**

## 4.3 确认策略创建成功

在 Policies 页面，你应该能看到 3 条策略都列在那里。如果是，图片仓库配置完成 ✅

# 第五步：把 Supabase 信息填到代码里

> 现在你需要把第二步记录的"项目地址"和"公钥"填到代码文件里。

## 5.1 在 GitHub 上编辑配置文件

1. 打开你的 GitHub 仓库页面（就是第一步上传代码的那个页面）
2. 点击 **js** 文件夹 → 点击 **config.js** 文件
3. 点击文件内容右上方的 **铅笔图标** ✏️（Edit this file）
4. 现在你可以编辑这个文件了

## 5.2 修改 4 个地方

在文件中找到以下内容并修改：

### 修改 1：SUPABASE_URL

找到这一行：
```
SUPABASE_URL: 'https://xxxxxxxx.supabase.co',
```
把引号里面的内容替换成**你记事本里的项目地址**。

例如改成：
```
SUPABASE_URL: 'https://abcdefgh.supabase.co',
```

### 修改 2：SUPABASE_ANON_KEY

找到这一行：
```
SUPABASE_ANON_KEY: 'eyJhbGciOiJ......很长一串......',
```
把引号里面的内容替换成**你记事本里的公钥**。

### 修改 3：ADMIN_PASSWORD

找到这一行：
```
ADMIN_PASSWORD: 'admin123',
```
把 `admin123` 改成**你想设置的管理密码**。

例如改成：
```
ADMIN_PASSWORD: 'wo2024de',
```

> 这个密码是你登录管理后台用的，设一个你能记住的就行。

### 修改 4：确认 DEMO_MODE 是 false

找到这一行：
```
DEMO_MODE: false,
```
确认它是 `false`。如果是 `true`，改成 `false`。

## 5.3 保存修改

1. 修改完成后，点击页面右上角的绿色按钮 **Commit changes...**
2. 在弹出的窗口中，直接点击 **Commit changes**
3. 修改保存成功 ✅

> ⚠️ **检查要点**：
> - 网址和密钥两边的**英文单引号 `'` 不要删掉**
> - 复制粘贴时注意**不要多出空格**
> - 每一行末尾的**逗号 `,` 不要删掉**

## 5.4 开启 GitHub Pages（发布网站）

> GitHub 自带免费的网站托管功能，叫 GitHub Pages。开启后你的代码就变成了一个可以访问的网站，不需要额外注册其他平台。

1. 在你的 GitHub 仓库页面，点击上方的 **Settings**（齿轮图标，在 Code / Issues / Pull requests 那一排的最右边）
2. 在左侧菜单中，找到 **Pages**，点击它
3. 在 "Build and deployment" 区域：
   - **Source**：选择 **Deploy from a branch**
   - **Branch**：选择 **main**，后面的文件夹选择 **/ (root)**
4. 点击 **Save**
5. 等待 1-2 分钟，页面上方会出现一个提示：
   ```
   Your site is live at https://你的用户名.github.io/product-showcase/
   ```
6. 点击这个链接，就能打开你的商品展示页面了！

> ⚠️ 第一次开启可能需要等 2-3 分钟才能访问，如果打开是 404，稍等一会儿再刷新。

## 5.5 记录你的网站地址

你的两个页面地址是：
- **商品展示页**（发给同事看的）：`https://你的用户名.github.io/product-showcase/`
- **管理后台**（你自己用的）：`https://你的用户名.github.io/product-showcase/admin.html`

> 把这两个地址记到记事本里！
>
> 以后每次你在 GitHub 上修改代码并保存，网站会在 1-2 分钟内自动更新。

# ❓ 遇到问题怎么办？

## 问题 1：管理后台登录后页面空白，什么都没有

**原因**：第五步填写的 Supabase 项目地址或公钥不对

**怎么办**：
1. 回到 Supabase → Settings → API，重新复制项目地址和公钥
2. 回到 GitHub，重新编辑 `js/config.js`，仔细粘贴
3. 保存后等 1-2 分钟，GitHub Pages 会自动更新，然后刷新网站页面

## 问题 2：添加商品时提示"图片上传失败"

**原因**：第四步的图片仓库没配置好

**怎么办**：
1. 回到 Supabase → Storage
2. 检查是否有 `product-images` 这个存储桶
3. 检查它是否是 Public 的（点击右边三个点 → Edit bucket → 确认 Public 开关是开的）
4. 检查 Policies 标签下是否有 3 条策略

## 问题 3：添加商品时提示"提交失败"

**原因**：第三步的数据库表没创建成功

**怎么办**：
1. 回到 Supabase → Table Editor
2. 看看是否有 `products` 和 `orders` 两个表
3. 如果没有，回到 SQL Editor 重新执行第三步的 SQL

## 问题 4：展示页面商品图片显示不出来（显示一个破碎的图标）

**原因**：图片仓库的 Public 开关没打开

**怎么办**：
1. Supabase → Storage → 点击 `product-images` 右边的三个点
2. 选择 **Edit bucket**
3. 把 **Public bucket** 开关打开
4. 保存

## 问题 5：点击"我要买"后提示"购买失败"

**原因**：第三步的第三段 SQL（购买功能）没执行成功

**怎么办**：
1. 回到 Supabase → SQL Editor → New query
2. 重新复制粘贴第 3.4 步的那段 SQL
3. 点击 Run 执行

## 问题 6：打开网站显示 404 页面

**原因**：GitHub Pages 还没生效，或者文件结构不对

**怎么办**：
1. 首先等 2-3 分钟再刷新试试（GitHub Pages 第一次部署需要一点时间）
2. 如果还是 404，打开你的 GitHub 仓库页面
3. 看看根目录下是否直接有 `index.html`
4. 如果看到的是一个 `product-showcase` 文件夹（里面才有 index.html），说明上传时多套了一层
5. 解决方法：删掉仓库重新来，上传时记得是把文件夹**里面的内容**拖进去
6. 另外检查 Settings → Pages 是否已经正确配置（Branch 选 main，文件夹选 root）

## 问题 7：修改了代码但网站没变化

**原因**：GitHub Pages 需要一点时间自动更新

**怎么办**：
- 在 GitHub 上保存修改后，等 1-2 分钟
